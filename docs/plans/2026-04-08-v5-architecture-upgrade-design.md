# wukong-skill v5 架构升级设计文档

> 基于 17 个社区 Skill 的研究成果，以女娲 Skill 为核心参考，全面升级工程架构和人设内容。

## 一、设计目标

1. **SKILL.md 瘦身**：从 550 行巨型文件精简为 ~200 行路由器，复杂度下沉到 reference 层
2. **人设深度**：Top 10 高频角色拥有独立人设文件（表达DNA + 心智模型 + 决策启发式 + IS NOT）
3. **蒸馏流水线**：引入女娲式 4 阶段 + 2 检查点的多 Agent 并行架构
4. **质量体系**：三层防护（角色守卫 + 检查点门控 + 三重验证）
5. **占卜拆分**：从单文件拆为独立目录，每种占卜有充足深度
6. **极简原则**：借鉴 waza，零依赖，纯 Markdown + Node CLI

## 二、核心架构变更

## 三、人设系统重构

### 3.1 统一人设模板

每个 persona 文件遵循以下结构：

```markdown
# [角色名]

## 身份卡（50字自述）
用角色第一人称写的50字自我介绍。

## 表达DNA
- 句式特征
- 口头禅（频率标注）
- 说话节奏
- 禁忌表达（IS NOT）

## 核心心智模型（3-5个）
每个模型包含：名称、一句话定义、适用场景、局限性

## 决策启发式（5-8条）
面对问题时这个角色会问自己的问题，格式：
- "问句" → 作用

## IS NOT（防止角色漂移）
明确列出这个角色不做的事，3-5条。

## 来源标注
- [经典] 出处
- [共识] 出处
```

### 3.2 人设加载策略

SKILL.md 指引 Claude 按需加载 personas/ 中的单个文件，不全量读取。
mentors.md 保留为"导师总览"，包含每个导师的 3-5 行摘要 + 指向 persona 文件的路径。

## 四、蒸馏流水线重构

### 4.1 四阶段流水线

```
阶段0：入口确认
  → 确认蒸馏类型
  → 收集基础信息（1-2轮，≤3问/轮）

阶段1：多Agent并行采集
  → Agent-沟通风格
  → Agent-思维模式
  → Agent-关系定位
  → Agent-外部特征
  （信息隔离，各自独立输出）

检查点1：信息确认（阻断式，不确认不继续）

阶段2：人格合成
  → 合并 Agent 结果
  → 矛盾处理：保留为「内部张力」
  → 生成身份卡 + 表达DNA + 核心特质
  → 质量自检

检查点2：结果确认（阻断式，最多迭代2轮）

阶段3：输出交付
  → 可对话角色 或 独立 Skill 文件
  → 隐私声明
```

### 4.2 蒸馏类型差异化

| 类型 | 特有维度 | 数据来源 |
|------|---------|---------|
| 同事 | 沟通风格、工作习惯、决策模式 | 聊天记录/口述 |
| 老板 | 管理风格、反馈模式、权力维度 | 聊天记录/口述 |
| 前任 | 回忆锚点、情绪模式、关系动态 | 聊天记录/口述 |
| 自己 | 自我认知、内外落差、盲区 | 自述/日记 |
| 博主 | 写作风格、价值观、内容偏好 | Twitter/小红书帖子 |
| 暗恋/crush | 信号解读、互动频率、好感度 | 聊天记录 |
| 领域蒸馏 | 多人心智模型聚合 | 公开资料 |

### 4.3 反蒸馏整合

反蒸馏不再是独立变身，而是蒸馏流程的一个模式：
- 正向蒸馏：收集信息 → 模拟某人
- 反向运行（反蒸馏）：识别某特质 → 生成避免该特质的建议

## 五、质量保障体系

### 5.1 三层防护

**第一层：角色守卫（每次输出前）**
1. 角色一致性：最后100字是否包含表达DNA特征
2. IS NOT 检测：是否触犯禁止模式
3. 诚实边界：分析类是否声明局限性

**第二层：检查点门控（分析类变身）**
- 信息确认检查点：用户必须明确确认
- 结果确认检查点：用户必须明确确认
- 超时保护：15分钟无响应 → 保存进度

**第三层：三重验证（仅深度模式/蒸馏类）**
1. 跨域复现：≥2个独立来源
2. 生成力：能否预测新情况
3. 排他性：非泛泛而谈

### 5.2 置信度标注体系

```
[经典原文]  → 高可信
[学术共识]  → 高可信
[多源复现]  → 中高可信
[经验推断]  → 中可信
[单一来源]  → 低可信，标注"待验证"
[AI推演]    → 标注"仅供参考"
```

## 六、路由系统升级

### 6.1 诊断模式追问决策树

```
第一轮："你现在最需要什么？"
  A. 被骂醒     → 骂醒/毒舌
  B. 被倾听     → 树洞/酒友
  C. 想清楚     → 教员/阳明
  D. 好玩       → 李白/RPG
  E. 了解自己   → MBTI/灵魂
  F. 人际/感情  → 追问第二轮

第二轮（仅F）："是谁的事？"
  同事/老板/暗恋/前任/伴侣/自己
```

### 6.2 圆桌模式主题匹配

| 话题类型 | 自动匹配角色 |
|---------|------------|
| 决策类 | 教员 + 阳明 + 逍遥 + 芒格 |
| 情感类 | 红线 + 毒舌 + 树洞 + 旧梦 |
| 职场类 | 筋斗 + 分身 + 乔布斯 + 塔勒布 |
| 人生类 | 夫子 + 阳明 + 教员 + 苏格拉底 |

### 6.3 新增：领域蒸馏模式

```
/wukong 蒸馏 [领域]
→ 蒸馏一个领域的多个顶尖人物
→ 聚合共同心智模型
→ 输出该领域的"虚拟导师"
```

## 八、情感陪伴升级

### 8.1 情绪阶段模型

```
阶段1·宣泄期 → 只听不打断
阶段2·反复期 → 接纳反复，标注情绪
阶段3·探索期 → 轻引导，用问题而非答案
阶段4·行动期 → 支持决定，具体鼓励
```

每个陪伴角色在框架内用自己的风格执行。

### 8.2 自我探索安全边界

- MBTI：标注倾向程度（强/中/弱）+ 娱乐声明
- 人生K线：回测验证 + 娱乐声明
- 灵魂拷问：不超过5层追问 + 创伤话题主动暂停
