# 悟空 / Wukong Skill

> **俺乃齐天大圣孙悟空。** 一个 Skill，七十二般变化。每一个变身都是一个人。

**其他语言 / Other Languages:** [English](README_EN.md)

**wukong-skill v5** 是一个 Claude Code 的超级 Skill，化身各种角色陪你——算命先生、孔夫子、教员、诗仙李白、你的暗恋对象……借鉴女娲Skill工厂的工程设计理念，采用多阶段流水线、人工检查点、三重验证方法论、多Agent并行架构，确保每次变身的输出质量。

**核心理念：好玩第一。** 不是工具合集，是角色切换器。

## 快速安装

### 方法一：推荐 — 使用 npx skills add

```bash
npx skills add Ming-H/wukong-skill
```

### 方法二：从 GitHub 克隆

```bash
git clone https://github.com/Ming-H/wukong-skill.git
cd wukong-skill
chmod +x install.sh && ./install.sh
```

## 效果示例

### 第十四变·教员

```
用户：我总感觉工作做不完，很焦虑

大圣：好！俺老孙变作「教员」，帮施主分析分析！

教员：这个问题嘛，要看到主要矛盾在哪里。
      你说工作做不完——是事情真的多，还是你没抓住重点？
      "眉毛胡子一把抓"可不行。

      先搞清楚：哪些是主要矛盾？哪些是次要的？
      把主要精力放在主要矛盾上，其他的可以先放一放。

      "没有调查，就没有发言权。"你先列出来，哪些事是非做不可的？

      前途是光明的，道路是曲折的。别急，一件一件来。
```

```
用户：就叫小A吧，我们是同事。ta喜欢喝手冲咖啡，说话很温柔……

暗恋：[分析中... 信号解读]
      回复速度指数：████░ (4/5)
      主动程度指数：██░░░ (2/5)

      建议：可以试试找机会跟ta聊咖啡——这是ta的兴趣点，
      看ta会不会主动延续话题。
```


## 设计理念（v5）

1. **全角色化**：每一个变身都是一个人。不是工具，是角色。
2. **好玩优先**：核心目标是好玩。有用是副产品。
3. **七十二变 = 统一接口**：不用安装17个Skill，一个孙悟空搞定。
4. **三通道智能路由**：直接变身 / 诊断推荐 / 增量更新。
5. **多阶段流水线 + 检查点**：借鉴女娲Skill工厂，信息确认和结果确认两个必经检查点。
6. **三重验证方法论**：跨域复现、生成力、排他性。
7. **诚实优于完美**：信息不足就标注、矛盾就保留、局限就声明。
8. **质量自检必执行**：6项质量自检，输出前不可跳过。
