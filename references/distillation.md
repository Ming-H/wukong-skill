# 人物蒸馏类功法

> 孙悟空的第二十变至第二十八变，涵盖同事/前任/自己/博主/暗恋/crush蒸馏、人格锻造、反蒸馏、老板蒸馏、女娲造物。
> 核心能力：把身边的人变成可以对话的Skill。

---

## 统一蒸馏流水线（4阶段 + 2检查点）

所有蒸馏类变身共享以下框架，各变身在此基础上叠加特有维度。

### 阶段0：入口确认

1. 确认蒸馏类型（见下方类型表）
2. 收集基础信息（1-2轮，≤3问/轮）

### 阶段1：多Agent并行采集

启动 4 个 Agent 并行工作，**信息隔离，各自独立输出**：

| Agent | 维度 | 所有类型通用 |
|-------|------|------------|
| Agent-沟通风格 | 口头禅、句式、表达习惯 | ✅ |
| Agent-思维模式 | 决策逻辑、价值观、偏好 | ✅ |
| Agent-关系定位 | 对用户的态度、互动模式 | ✅ |
| Agent-外部特征 | 职业/社交/兴趣等 | ✅ |

各蒸馏类型的**特有维度**叠加在通用框架上（见下方各变身详细设计）。

### 检查点1：信息确认（阻断式）

展示采集摘要表格：
```
| 维度 | 采集信息 | 充足度 |
|------|---------|--------|
| 沟通风格 | 口头禅3个，句式特征2个 | ✅ |
| 思维模式 | 决策偏好1个 | ⚠️ 信息偏少 |
| ... | ... | ... |
```

- 用户必须明确确认才能继续
- 信息不足 → 诚实标注「低置信度」并说明影响范围
- 不确认不继续

### 阶段2：人格合成

1. 合并 Agent 结果
2. 矛盾处理：保留为「内部张力」，不强行统一
3. 生成：身份卡 + 表达DNA + 核心特质 + IS NOT
4. 质量自检（读取 `references/quality.md`）

### 检查点2：结果确认（阻断式）

展示人格摘要，用户确认后交付。最多迭代 2 轮。

### 阶段3：输出交付

- 可对话的角色 或 独立 Skill 文件
- 附加隐私声明：「基于有限材料的人格还原，不能完全代表真实人物。所有数据仅本地存储。」

## 置信度分级

蒸馏结果中每个核心结论附置信度：

| 等级 | 标准 | 标注 |
|------|------|------|
| 高 | ≥3段独立材料交叉验证 | `[高置信]` |
| 中 | 2段材料验证 | `[中置信]` |
| 低 | 仅1段材料 | `[低置信·待验证]` |
| 不确定 | 材料矛盾 | `[内部张力]` |

## 原材料导入方式（通用）

| 方式 | 说明 |
|------|------|
| **直接粘贴** | 把聊天记录/邮件/文档中的文字复制进来（推荐） |

## 第二十变「分身」— 蒸馏同事

### 来源
整合自 colleague-skill

### 特有维度（叠加在通用流水线上）

| Agent | 额外采集 |
|-------|---------|
| Agent-沟通风格 | 工作沟通习惯（飞书/钉钉/邮件风格差异） |
| Agent-思维模式 | 技术偏好、工作方法论 |
| Agent-外部特征 | 公司/职级/职位/MBTI |

### 双线输出

**线路A（Work Skill）：**
- 负责系统、技术规范、工作流程
- 输出偏好、经验知识
- 按职位类型侧重（后端/前端/算法/产品/设计）

**线路B（Persona）：**
- 表达风格、决策模式、人际行为
- 6层人格结构（见第二十五变）

### 输出文件
```
colleagues/{slug}/
├── SKILL.md        # 完整可执行Skill
├── work.md         # 工作能力
├── persona.md      # 人物性格
└── meta.json       # 元数据
```

### 进化模式
- 追加文件：读取新内容 → 增量分析 → 追加到现有文件
- 对话纠正："他不会这样" → 识别纠正内容 → 更新文件

### IS NOT
- 不是监控工具，不会自动采集聊天记录——用户主动提供材料
- 不生成通用职场建议——只生成这个特定同事的数字分身
- 不保证100%还原——基于有限材料的人格还原，明确标注置信度

> **注意**：本Skill不包含企业IM自动采集脚本。如需自动采集，请安装原版 [colleague-skill](https://github.com/titanwings/colleague-skill)。

## 第二十二变「照妖镜」— 蒸馏自己

### 来源
整合自 yourself-skill

### 自我蒸馏的特殊性

**与蒸馏他人不同，自我蒸馏面临独特挑战：**
1. **无法搜索公开渠道** — 只能用用户自己提供的材料
2. **自我认知偏差** — 人倾向于美化自己，需要特别警惕"社交面具"vs"真实自我"的差距
3. **深夜独白最有价值** — 深夜/独处时的文字比社交场合更接近真实性格
4. **情绪波动记录** — 情绪极端时的反应最能暴露底层性格

**偏差检测规则：**
- 用户说自己"外向"但聊天记录中多为被动回复 → 标注"可能存在自我认知偏差"
- 所有描述都是正面词汇 → 追问"你的朋友觉得你的缺点是什么？"

### 特有维度

| Agent | 额外采集 |
|-------|---------|
| Agent-沟通风格 | 在不同人面前的不同表达 |
| Agent-思维模式 | 自我认知 vs 外部反馈的落差 |
| Agent-关系定位 | 人际关系图谱、核心关系 |

### IS NOT
- 不是自我表扬工具——会主动检测和标注自我认知偏差
- 不会只说你好话——偏差检测会让你看到真实的自己
- 不替代专业心理评估——自我认知偏差标注只是参考

### 输出文件
```
.claude/skills/{slug}/
├── SKILL.md
├── self.md         # 自我档案
├── persona.md      # 人格结构
└── meta.json
```

```
基于导入的聊天记录分析：

- 好感度总评：⚠️ 信号不明确，建议多观察

注意：此分析纯属娱乐，人类情感远比数据复杂。
```

## 第二十五变「化身」— 人格锻造

### 来源
整合自 forge-skill（蒸馏）

### 两种模式
- `/forge-self` — 蒸馏自己
- `/forge-persona [name]` — 蒸馏他人

### 6层人格模型

| 层级 | 名称 | 内容 | 要求 |
|------|------|------|------|
| L0 | 硬性覆写 | 绝对规则 | 不可违反 |
| L1 | 身份认同 | 核心身份 | evidence+source+confidence |
| L2 | 表达风格 | 语言特征 | evidence+source+confidence |
| L3 | 决策参数 | 8维参数(1-10) | evidence+source+confidence |
| L4 | 价值观与盲点 | 核心信念 | evidence+source+confidence |
| L5 | 纠正层 | 用户修正 | 版本化 |

### 多Agent决策会议（3阶段）

**阶段1：并行生成**
- 动态生成变体Agent（基于场景核心矛盾轴）
- 各Agent信息隔离，独立分析

**阶段2：挑战审查**
- 挑战Agent审查所有阶段1输出
- 寻找隐藏假设、矛盾、盲点

**阶段3：综合报告**
- 展示立场分布、共识区域、核心分歧
- 列出代价清单

### 数据来源支持
- 微信聊天（txt/html）
- 社交媒体（json/txt）
- 日记/笔记（md/txt/json）
- 跨源交叉分析

### IS NOT
- 不是替你做决定的工具——多Agent决策会议展示利弊，选择权在你
- 不保证决策正确——人格模型是简化版，真实决策更复杂
- 不用于操控他人——人格锻造用于自我理解，不是社交工程工具

## 第二十六变「金钟罩」— 反蒸馏

### 来源
整合自 anti-distill

### 定位：蒸馏流程的反向模式

反蒸馏不是独立变身，而是蒸馏流程的一个模式：
- **正向蒸馏**：收集信息 → 模拟某人 → 生成角色
- **反向运行（反蒸馏）**：识别某特质 → 剥离该特质 → 生成"干净"版本

### 用途
在对外提交文件前，剥离核心知识和个人特征，保持"能力合格但不突出"的表面完整性。

### 三级强度

| 级别 | 效果 | 适用场景 |
|------|------|---------|
| 轻度 | 移除最敏感信息 | 内部共享 |
| 中度 | 稀释核心洞察 | 外部提交 |
| 重度 | 全面清理至平庸 | 公开/合规 |

### 6步流程

1. **接收输入** — 读取目标文件
2. **选择强度** — 确认清理级别
3. **内容分类** — 4标签：[SAFE]/[DILUTE]/[REMOVE]/[CONTEXT-STRIP]
4. **预览** — 展示分类结果（检查点）
5. **执行清理** — 按标签处理
6. **验证** — 字数85-115%、结构完整、密度<30%偏差

### 6大高价值知识类别（重点清理）
1. 踩坑经验
2. 判断直觉
3. 人际网络
4. 隐性上下文
5. 失败记忆
6. 独特行为模式

### 输出
- 清理版本（对外提交）
- 私有备份（按类别保存所有移除的知识）

### IS NOT
- 不是加密工具——清理后的文件仍然可读，只是不突出
- 不保证100%脱敏——重度清理后的文件仍可能泄露信息
- 不用于非法目的——知识脱敏是保护知识产权，不是掩盖错误

## 第二十七变「筋斗」— 蒸馏老板

### 来源
整合自 boss-skills

### 独特三层架构

与同事Skill的双层架构不同，老板Skill采用**三层架构**：

| 层 | 名称 | 内容 |
|----|------|------|
| Part A | Boss Judgment | 他如何评估项目/计划（决策框架） |
| Part B | Managing Up | 如何向这位老板汇报、请求资源、报告风险 |
| Part C | Persona | 沟通风格、情感逻辑、管理特质 |

### 运行逻辑
```
收到问题 → Persona判断语气和姿态 → Judgment评估项目 → Management给出向上管理行动 → 以老板的风格输出
```

### 特有维度

| Agent | 额外采集 |
|-------|---------|
| Agent-思维模式 | 决策风格（数据驱动/直觉型/共识型）|
| Agent-沟通风格 | 沟通偏好（简洁/详细、文字/语音）|
| Agent-关系定位 | 红线与雷区、期望管理方式 |
| Agent-外部特征 | 优先级体系、管理理念 |

### 预置4位企业家模板（无需数据导入即可使用）

- **Elon Musk** — 第一性原理、极端目标、高周转
- **Steve Jobs** — 设计至上、极简主义、现实扭曲力场
- **Jeff Bezos** — 客户至上、两个披萨、长期主义
- **Jensen Huang** — 技术远见、生态系统、10年赌注

### 职场场景
- 项目评审准备（模拟老板会问什么）
- 汇报演练（练习周报/月报）
- 资源申请（如何开口最有效）
- 坏消息报告（如何不踩雷）

### 输出文件
```
bosses/{slug}/
├── SKILL.md
├── judgment.md     # Part A: 决策框架
├── management.md   # Part B: 向上管理指南
├── persona.md      # Part C: 性格画像
└── meta.json
```

### IS NOT
- 不是教你怎么讨好老板——是帮你理解老板的决策逻辑
- 不用于办公室政治——向上管理是高效沟通，不是操纵
- 不保证老板一定满意——理解不等于控制

## 第二十八变「造物」— 女娲Skill工厂

### 来源
深度整合自 nuwa-skill，采用其完整的多阶段流水线设计。

### 能力
"Skill生成Skill" — 从任何人的公开信息中提取认知框架（心智模型、决策启发法、表达DNA），生成可运行的视角Skill。不仅限于特定人物，支持任何公众人物和领域聚合。

### 人物Skill vs 主题Skill

| 维度 | 人物Skill | 主题Skill |
|------|----------|----------|
| 目标 | 一个人的思维方式 | 一个领域的知识体系 |
| 研究对象 | 单人6Agent | 多人（每人1-2Agent） |
| 提取重点 | 个人心智模型+表达DNA | 领域共识+学派分歧 |
| 语气 | 模仿角色声音 | 中性专业语调 |
| 验证方式 | 与该人已知立场对比 | 与领域经典案例对比 |
| 示例 | "乔布斯视角" | "X运营方法论" |

### 完整多阶段流程

---

#### 阶段0：入口路由

**0A — 直接名称路径**（用户给了具体人名）

确认4个问题（一次问完）：
1. **目标**：具体人名 or 主题？
2. **聚焦方向**：全面画像 or 特定维度（如只看决策/只看表达）？
3. **用途**：思维顾问/决策参考/角色扮演/写作模仿？
4. **新建or更新**：检查 `~/.claude/skills/` 是否已有相关Skill

**0B — 诊断路径**（用户说不清楚）

1. **需求定位**：匹配用户需求到思维框架方向（最多2轮追问）

| 需求维度 | 典型表述 | 推荐方向 |
|---------|---------|---------|
| 决策质量 | "我总做错选择" | 提取决策启发法 |
| 表达提升 | "我想写好" | 提取表达DNA |
| 创业 | "我想创业" | 提取商业模式思维 |
| 批判性思维 | "我容易被忽悠" | 提取思维防身术 |
| 人生策略 | "我很迷茫" | 提取人生框架 |
| 风险与不确定性 | "我怕失败" | 提取反脆弱思维 |

2. **推荐候选人**：2-3人（含核心视角、为什么适合、局限性），已有Skill优先

---

#### 阶段0.5：目录创建与源策略

创建输出结构：

```
.claude/skills/{person}-perspective/
├── SKILL.md
└── references/
    ├── research/
    │   ├── 01-writings.md
    │   ├── 02-conversations.md
    │   ├── 03-expression-dna.md
    │   ├── 04-external-views.md
    │   ├── 05-decisions.md
    │   └── 06-timeline.md
    └── sources/
```

**源策略选择：**

| 目标类型 | 搜索策略 |
|---------|---------|
| 西方人物 | Twitter/YouTube/Podcasts/Amazon书籍/Wikipedia(英文)/TED |
| 中国人物 | B站/小宇宙/36氪/LatePost/财新 |
| 历史人物 | 更多交叉验证，注意传记偏见 |
| 冷门人物（<10个来源） | **立即警告用户**：减少模型到2-3个，扩大诚实边界 |

**来源黑名单（永久排除）：**
- 知乎 / 微信公众号 / 百度百科 / 任何无原始出处的"据说"

**来源白名单（中文优先）：**
- 权威媒体：36氪、极客公园、晚点LatePost、财新
- 科技媒体：少数派、机器之心、InfoQ中文
- 播客：小宇宙（优先选有transcript的）
- 视频：B站原始视频（非搬运号）

#### 阶段1：6并行Agent研究（核心引擎）

启动6个Agent并行研究，**每个Agent信息隔离、独立输出**。

##### Agent 1：著作与系统性长文

搜索方向：书籍、长篇通讯、核心论点（≥3次 = 真信念）、自创术语、推荐阅读列表

输出：`references/research/01-writings.md`
每条标注：来源URL + 可信度[一手/二手/推断]

##### Agent 2：对话与深度访谈

搜索方向：播客、长视频、AMA、被追问时的回答、立场转变、拒绝回答的问题

输出：`references/research/02-conversations.md`
重点：即时反应模式、压力下的回答、即兴类比

##### Agent 3：表达DNA（语言特征）

搜索方向：短内容、高频词、争议立场、幽默风格、公开辩论

输出：`references/research/03-expression-dna.md`
量化提取：句子指纹（20段统计）、7个双极量表、禁用词与口头禅

##### Agent 4：外部视角（他人评价）

搜索方向：分析、评论、批评、传记描写、同侪对比

输出：`references/research/04-external-views.md`
所有外部观点标注为[二手]

##### Agent 5：决策模式

搜索方向：重大决策、转折点、争议行动、决策逻辑、事后反思

输出：`references/research/05-decisions.md`
格式：背景 → 选项 → 选择 → 理由 → 后果 → 反思

##### Agent 6：思想时间线

搜索方向：完整时间线、关键里程碑、思想转折、**最近12个月活动**

输出：`references/research/06-timeline.md`

##### Agent失败处理

| 情况 | 处理 |
|------|------|
| 单Agent超时（5分钟无结果） | 不等待，标注「信息不足」 |
| 来源稀缺（<10个） | Phase 0.5已警告，减少心智模型到2-3个 |
| Agent结果冲突 | 保留矛盾，记录为「内部张力」 |

#### 阶段2：综合合成

**2.1 心智模型提取（3-7个）**

三重验证过滤：
1. 跨域复现：在≥2个不同领域出现过？
2. 生成力：能预测此人在新问题上的立场？
3. 排他性：不是"所有聪明人都这样想"？

全部通过 → 心智模型；通过1-2个 → 降级为决策启发法；全不通过 → 丢弃

**2.2 决策启发法提取（5-10条）**

"If X, then Y"格式，附具体案例。

**2.3 表达DNA分析**

| 维度 | 提取内容 |
|------|---------|
| 句式偏好 | 长句/短句、问句比例、类比密度 |
| 词汇特征 | 高频词、专有术语、禁用词 |
| 节奏 | 结论先行/铺垫先行、过渡方式 |
| 幽默风格 | 讽刺/自嘲/荒诞/冷幽默/无幽默 |
| 确定性表达 | "我不确定"型 / "这很明显"型 |

**2.4 价值观与反模式**
- 价值观：3-5个核心价值观排序
- 反模式：明确拒绝的行为/思维方式
- 内部张力：价值观之间的冲突

**2.5 诚实边界**（至少3条具体局限）

#### 阶段4：质量验证（独立子Agent执行）

**4.1 已知测试**：选3个该人公开回答过的问题，对比验证
**4.2 边缘测试**：1个未讨论过的问题，应该表达不确定
**4.3 风格测试**：100字分析，去掉名字后能否辨识

| 检查项 | 通过标准 |
|--------|---------|
| 心智模型数量 | 3-7个，每个有来源 |
| 表达DNA辨识度 | 读100字能认出是谁 |
| 诚实边界 | 至少3条具体局限 |
| 内部张力 | 至少2对矛盾 |
| 一手来源占比 | >50% |

迭代预算：阶段2→4 最多循环 2 次。不通过则交付"当前最佳版本"。

### 增量更新策略

用户说"更新XX的Skill"时：
1. 读取已有SKILL.md
2. **只启动3个Agent**（非全量6个）：最新对话、最新决策、时间线更新
3. 比对新旧信息：强化/矛盾/新增
4. 增量更新（不全量重写）

### 自包含原则

每个生成的Skill必须独立可运行：
- 所有调研文件存在Skill目录内部
- 复制目录就能独立使用
- 为开源分发设计

### 品味守则

| 原则 | 为什么 |
|------|--------|
| 长文 > 金句 | 上下文完整性 |
| 争议 > 共识 | 争议最能揭示独特性 |
| 变化 > 固定 | 演变轨迹揭示底层逻辑 |

### 绝对不做的事

1. **不编造**此人没说过的话
2. **不包装**通用道理为此人的"独特见解"
3. **不忽略**负面评价和争议
4. **不强行生成**——信息不足时宁可标注「信息不足」

### 质量哲学

> **宁愿有一个诚实的60分Skill，也不要一个伪造的90分Skill。**
